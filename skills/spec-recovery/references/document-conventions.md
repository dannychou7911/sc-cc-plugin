# 文件通用規範

所有 L0–L4 產出文件必須遵守以下規範。

---

## Header

每份文件開頭包含：

```markdown
# {文件標題}

> **層次**：L{0-4}
> **最後更新**：{日期}
> **狀態**：初版 / 已驗證 / 待更新
> **負責人**：{姓名或留空}
```

## 目錄（TOC）

Header 之後、正文之前，必須包含**目錄區塊**，列出所有一級和二級標題的連結。

```markdown
## 目錄

- [1. 章節名稱](#1-章節名稱)
  - [1.1 子章節](#11-子章節)
- [2. 章節名稱](#2-章節名稱)
```

## Confidence Markers

| 標記 | 意義 |
|------|------|
| `[確認]` | 程式碼明確顯示 |
| `[推斷]` | 合理推論 |
| `[待確認]` | 需人工確認 |
| `[未確定]` | 問過使用者但無法確定，列入 Open Questions |
| `[未確定-業務脈絡]` | 業務知識缺失，非程式碼分析不足 |

## 流程圖

一律使用 **mermaid.js flowchart 語法**繪製，不使用純文字樹狀圖。適用於 L0–L4 所有層次。

## 隱性規則標記

使用 🔴/🟡 分級標記。分級標準詳見 `implicit-rules-standard.md`。

## 語言

- 預設使用繁體中文撰寫
- 技術名詞中英文並行
- 跟隨使用者的語言偏好
