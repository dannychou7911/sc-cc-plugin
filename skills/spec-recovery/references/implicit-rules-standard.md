# 隱性規則分級標準

## 分級定義

| 標記 | 名稱 | 定義 | 判斷標準 |
|------|------|------|---------|
| 🔴 | 重構必知 | 忽略會直接導致 bug 或功能失效 | 移除或修改此規則後，現有功能會壞掉 |
| 🟡 | 資訊性 | 有助理解但忽略不會立即出錯 | 移除或修改此規則後，功能仍正常，但可能影響效能、可維護性或使用者體驗 |

## 分級判斷流程

```
發現一個隱性規則
  ├── 問：如果有人在重構時忽略了它，會不會出 bug？
  │   ├── 會 → 🔴
  │   └── 不會 → 🟡
```

## 各類別典型歸類

### 🔴 重構必知（忽略 = bug）

| 類別 | 範例 |
|------|------|
| 位元運算 / 狀態判斷 | `status & 0x08` 判斷特定狀態旗標 |
| 錯誤碼映射 | 內部錯誤碼轉換為使用者可見的錯誤碼 |
| 前後端隱性契約 | API 回傳欄位名與前端使用名不同（需重命名） |
| 權限相依性 | 選中某權限自動連帶添加其他權限 |
| 資料格式轉換 | 密碼加密後傳送（如 SHA-1、bcrypt） |
| Token 中間狀態 | 認證流程中某些 token 只是暫時性的，功能受限 |
| 非同步呼叫順序依賴 | 必須先查 A 再查 B，順序不可互換 |
| Magic value（影響邏輯分支） | `type === 11` 決定不同的業務邏輯路徑 |
| 靜默截斷 / 精度處理 | `.slice()` / `.toFixed()` 截斷使用者不知道 |
| 固定格式的 ID / 前綴 | ID 由固定前綴 + 動態部分組成 |

### 🟡 資訊性（有助理解但不影響正確性）

| 類別 | 範例 |
|------|------|
| 分頁大小 | `pageSize = 50` |
| 外部服務依賴 | 使用第三方 API 生成圖片或發送通知 |
| 硬編碼上限 | `maxAmount = 999999` |
| UI 行為細節 | 密碼欄位 focus 時清空、自動焦點轉移 |
| 排序方式 | 前端 `localeCompare()` 排序 |
| 響應式斷點 | 視窗寬度 ≤ 768px 取消欄位固定 |
| 快取清除時機 | 操作成功後清除特定快取鍵 |

## 在文件中的格式

```markdown
## N. 隱性規則

🔴 **密碼加密**：所有密碼透過加密函式處理後傳送，前端不存明文

🔴 **Token 中間狀態**：某些認證步驟回傳的 token 功能受限，不可用於一般 API 呼叫

🟡 **pageSize = 50**：硬編碼分頁大小

🟡 **第三方圖片服務**：使用外部 API 生成特定圖片（如 QR Code）
```

## 在 Agent prompt 中的指示

探索 agent 回傳隱性規則時，應使用以下格式：

```
## 隱性規則
- 🔴 {規則描述}（忽略會導致：{後果}）
- 🟡 {規則描述}
```
