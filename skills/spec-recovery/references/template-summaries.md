# 模板格式要求摘要

> 供主 agent 傳給 `spec-scorer` agent 時使用，避免每次重讀完整模板。
> 若模板有更新，需同步更新本檔案。

---

## L0 格式要求

- 必須包含：Header、TOC、專案基本資訊、技術棧、模組清單與依賴關係圖（mermaid）、系統級決策考古
- 模組清單用表格（模組名、職責、入口點數、備註）
- 依賴關係用 mermaid flowchart
- Confidence Tagging 標記所有發現項
- 附錄：既有文件參考清單

## L1 格式要求

- 必須包含：Header、TOC、模組職責、模組邊界（負責/不負責）、資料流（mermaid）、對外依賴、設計決策考古、隱性規則（🔴/🟡 分級附 Why）
- 狀態機用 mermaid stateDiagram
- 同名異義概念標記
- 依賴關係用表格（依賴模組、依賴方式、方向）

## L2 格式要求

- 必須包含：Header、TOC、進入條件、初始載入行為（含外部呼叫 + 成功/失敗路徑）、使用者操作或觸發動作（逐一列舉）、條件邏輯規則、錯誤處理、隱性規則（🔴/🟡 分級附 Why）
- 流程使用 mermaid flowchart
- 若有表單：欄位清單表格（欄位名、型別、必填、驗證規則、預設值）、送出流程、離開防護
- 可復用邏輯必須展開完整行為
- 共用邏輯（≥2 模組使用 + 非平凡行為）獨立建檔

## L3 格式要求

- 必須包含：Header、TOC、Base URL、認證方式、共通錯誤回應格式
- 每支 API：方法、路徑、認證需求、角色限制、Request 參數表格、Request 範例、Response 格式、錯誤碼表格
- 隱性契約標記（命名轉換、分頁機制、日期格式、數值精度等）
- 隱性規則（🔴/🟡 分級附 Why）

## L4 格式要求

- 必須包含：Header、TOC、需要註解的程式碼段落清單、建議標記（TODO/FIXME/HACK/NOTE/@see/@since）、測試接縫建議、活文件維護策略
- 註解僅記錄 Why 和 What for，不記錄 What
- 隱性規則彙總（🔴/🟡 分級附 Why）
